# 05. 화면 갭 리포트 (계획 vs 실제 구현)

> 임의 수정 없이 보고. 이번 작업에서 코드를 변경하지 않음.

## 1. 계획과 실제 화면의 차이

| 항목 | 기존 계획 가정 | 실제 구현 | 영향 |
|---|---|---|---|
| 로그인 후 첫 화면 | 학생 대시보드 | `/student/mypage` | 3쪽 원고를 마이페이지 기준으로 수정 |
| 핵심어 위치 | 단원 선택에 포함 가능 | `/student/topic`(주제 만들기)의 서브 단계 | 6쪽을 주제 만들기 아래로 이동 |
| 생활 속 이야기 | 별도 단계 | `단원 정리`의 “세상 속 이야기” → `생활 연결` 탭 | 16쪽 제목을 “세상 속 이야기 만들기”로 수정 |
| OX 퀴즈 | 별도 단계(`/student/quiz/intro`) | `단원 정리`의 “OX 문제 5개 만들기” | 17쪽을 단원 정리 페이지 기준으로 작성 |
| 좌측 단계 표시 | 미확정 | `StudentFlowSidebar` 8단계(단원선택→…→만화보기) | 4쪽 흐름도 확정 |

## 2. 버튼 문구 불일치 / 확인 항목
- “주제 만들기 ✨”, “대본 만들기 🚀”, “표지만들기”, “만화 만들기”, “단원 정리 →”, “뒤표지 만들기”, “만화 보기 🖼️”, “책 펼치기”, “친구에게 자랑하기”, “링크 복사하기” — 코드 확인값. 원고에 동일 문구 사용.
- 만화제작 도구: `선택` `캐릭터` `대사` `말풍선` `배경` `레이어`.
- 대본 도구: `AI 생성` `컷 편집` `핵심 개념` `표지 대화`.
- 단원선택 1·2단계 버튼: `다음 단계 🚀`, `주제 만들기 ✨`.

## 3. 화면에서 찾기 어려운 기능
- 만화제작 배경 재생성: “배경 수정은 컷마다 1번만 할 수 있어요.”(제한) — 12쪽 주의사항에 명시.
- 핵심어 최대 4개 선택 제한 — 6쪽에 명시.
- 주제 최대 10개 — 7쪽에 명시.

## 4. 캡처하기 어려운 화면
- 만화보기 플립북 넘김 애니메이션 — 정적 캡처로 대체.
- AI 생성 중 로딩 화면 — 타이밍 캡처 필요.
- 공유 링크 생성(html2canvas 베이킹) — 링크 완료 모달로 대체.

## 5. 구현되지 않은 기능 (설명서 제외)
- `/student`(StudentDashboard): 목 데이터, `/toon` 레거시 연결. 로그인 후 도달 안 됨. → 제외.
- `/student/quiz/intro`(마무리 퀴즈): 메인 흐름 아님. → 제외.
- `/student/comic/cut/:cutNumber`: 메인 흐름 미사용. → 제외.
- 만화제작 Undo/Redo: `disabled`(비활성). → 설명서에 기재하지 않거나 “준비 중” 명시.
- 로그인 “아이디/비밀번호 찾기”: 핸들러 없음. → 제외.
- 뷰어 “썸네일 보기”: “준비 중” 안내. → 제외.

## 6. 사용자 혼동 가능성
- 마이페이지에서 미완성 작품 “이어하기” 클릭 시 `?projectId=` 가 무시되어 새 단원 선택으로 시작됨(`StudentUnitSelectPage`가 쿼리 파라미터 미읽기). → 19쪽에 “이어하기는 projectId 전달 시에만 동작” 주의사항 추가 권장. 단, 코드 수정은 이번 작업에서 안 함.
- 단원선택 1단계 뒤로가기가 `/student/dashboard`(레거시)로 이동. → 5쪽에서 “뒤로는 마이페이지로” 안내 시 실제와 다를 수 있음(주의).

## 7. 설명서 제작 전 수정 필요 사항 (권장만, 미수정)
- 마이페이지 이어하기 기능 정상화(`?projectId=` 읽기).
- 단원선택 뒤로가기 경로를 마이페이지로 통일.
- 위는 별도 이슈로 보고하며 이번 설명서 작업에서는 코드 변경하지 않음.
