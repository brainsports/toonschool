# 04. 브라우저 화면 캡처 절차

> 본 절차는 실제 촬영 시 사용. 현재 환경에서는 테스트 계정·브라우저 자동화가 없어 **미실행**. 코드 조사로 화면 구성은 확인 완료.

## 1. 테스트 계정 준비
- 테스트 학생 1명 생성(선생님 계정으로 `create-student` EF 사용, 또는 관리자 화면). 권장 정보:
  - 이름: 툰스쿨 학생 / 학년: 초등 4학년 / 반: 1반
  - 과목: 과학 / 학기: 1학기 / 단원: 실제 `curriculum_units` 에 존재하는 4학년 1학기 과학 단원
  - 작품 주제: 물의 여행 / 작품 제목: 물방울의 신나는 여행
- 실제 시스템에 존재하는 학년·학기·과목·단원만 사용. 없으면 가장 가까운 값으로 변경 후 원고에 동일 적용.
- 운영 계정·실제 학생 데이터 사용 금지.

## 2. 테스트 작품 제작 순서(캡처용)
1. `/login?mode=student` 로그인 → `/student/mypage`.
2. `툰스쿨 에디터 입장` → `/student/select-unit`.
3. 학년(초4)·학기(1학기) → 과목(과학)·대단원·중단원 선택.
4. 주제 만들기: 키워드 선택 → 주제 생성 → “물의 여행” 계열 주제 선택.
5. 대본 만들기: `AI 생성하기` → 6컷 대본 + 핵심개념3 + 표지대화3.
6. 표지만들기: `표지 만들기` → `만화 만들기`.
7. 만화제작: `배경 모두 생성` → 캐릭터/말풍선 배치 → 6컷 완료 → `단원 정리 →`.
8. 단원 정리: 세상 속 이야기 만들기 → OX 문제 5개 만들기 → `뒤표지 만들기`.
9. 뒤표지: 정보 입력·색상 → `만화 보기 🖼️`.
10. 만화보기: `책 펼치기` → 넘기기 → `친구에게 자랑하기` → 링크 복사.
11. 마이페이지에서 작품 목록 확인. `/book/:slug` 공유 링크 열람 확인.

## 3. 브라우저 설정
- Chrome, 1920×1080, 확대 100%.
- 개발자 도구 닫기, 북마크 표시줄 숨기기, 불필요한 탭/알림/다운로드창 닫기.
- 마우스 커서가 버튼 글자를 가리지 않도록 배치.
- 저장 형식: PNG(원본 비율 유지).

## 4. 단계별 저장 지점(캡처 타이밍)
- 각 단계마다 “생성 전 / 생성 중 / 결과” 상태를 캡처(`03-screenshot-checklist.md` 파일명 사용).
- 데이터 초기화 금지: 한 테스트 작품을 끝까지 유지. 중간에 새로고침하면 localStorage(`toonschool:*`) 유지로 이어짐(단, projectId 일치 필요).
- 비밀번호 입력 상태 캡처 금지. 이메일/ID/실제 이름 마스킹.

## 5. 캡처 확인 기준
- 버튼 글자가 읽힐 해상도.
- 강조할 요소가 화면 안에 포함.
- 개인정보·타인 데이터 미노출.

## 6. 이미지 정리
- 원본: `screenshots/originals/`.
- 설명서용(크롭/리사이즈): `screenshots/selected/`. 동일 파일명 유지.
- 번호/화살표 합성 금지. PPTX 도형으로 표시.

## 7. 현재 한계
- 본 환경(CLI)에서 브라우저 자동화(Playwright 등) 미설치, 테스트 계정 미보유.
- → 캡처는 위 절차에 따라 별도 환경에서 수행. PPTX/DOCX에는 해당 파일명의 **플레이스홀더 이미지 프레임**을 두어 실제 캡처 교체만으로 완성되도록 설계.
